TDD와 단위테스트는 다른이야기

## TDD
  테스트가 주도하는 개발, 테스트 코드를 먼저 작성
### Red green cycle
1. 항상 실패하는 테스트를 먼저 작성하고 red
2. 테스트가 통과하는 프로덕션 코드를 작성하고 green
3. 테스트가 통과하면 프로덕션 코드를 리펙토링 refactor

## 단위테스트
TDD의 첫번쨰 단계인 기능 단위의 테스트 코드를 작성, 테스트코드를 먼저 작성해야 하느것도 아니고 리펙토링도 포함되지 않음
